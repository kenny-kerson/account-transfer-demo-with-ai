# ADR-0007: 완료 응답·수취조회·알림·다건 규모 (Round 4)

- Status: Accepted
- Date: 2026-05-23
- Deciders: kenny.k
- Tags: behavior, operations

## Context

24/7 실시간 + 외부 위임 구조에서 발생하는 운영 디테일.

## Decision

### 타행이체 완료 응답 시점
- **동기 — 수신은행 입금 확인 후 완료 응답**
- 사용자 입장에서 "이체 완료" 시점이 가장 명확
- 타임아웃·재시도·SLA 설계가 핵심 과제가 됨

### 수취인 사전검증 (PreflightCheck)
- **필수** — 예금주명 확인뿐 아니라:
  - 계좌 상태 (정상/지급정지/해지)
  - 잔액 수취 가능 여부
  - 기타 선검증 일체화
- 결과는 PayeeVerificationSnapshot 엔터티로 보관 (ADR-0009 카테고리 A)

### 알림
- **외부 알림 시스템에 위임** (ADR-0003)
- 이체 서비스는 클라이언트 구현만 (이벤트 발행 → 알림 시스템 호출)

### 다건이체 상한
- **대규모 배치 — 1회 최대 약 10,000건 이상**
- 비동기 처리, 진행률 추적, 부분 재실행 설계 필수

## Rationale

- 동기 완료 응답: 사용자 UX 명확성 + 시연 가치 (타임아웃 처리·재시도·서킷 브레이커 패턴이 자연스럽게 정당화)
- 사전검증 필수: 착오송금 예방 + 한국 금융권 표준 (보이스피싱 대응)
- 다건 10,000건+: 시니어 데모로서 배치 처리·청크 분할·진행률 추적·실패 격리 패턴 시연

## Alternatives Considered

- **완료 응답 비동기 (출금 시점 응답, 입금 결과는 푸시)** — 거부됨. 사용자 UX 모호, 시연 가치는 동기 모델이 더 풍부
- **수취 조회 선택적** — 거부됨. 착오송금 리스크
- **다건 100건/1000건 상한** — 거부됨. 시연 가치 부족, 청크·진행률·재실행 패턴 시연 약함

## Consequences

- (+) 동기 응답 → 타임아웃·서킷 브레이커·재시도 시연 정당화
- (+) PreflightCheck → 도메인적으로 명확한 별도 단계, 외부 시스템 호출 클라이언트 추가 시연
- (+) 10,000건+ → 비동기 배치·진행률·DLQ·부분 재실행 시연 풍부
- (-) 동기 모델은 타행망 SLA에 본 시스템 SLA가 종속 (NFR P99 300ms 달성 어려움 → 외부 위임 SLA 가정 + 백프레셔 패턴 필요)
- (-) 다건 10,000건+ → 단일 요청·트랜잭션으로는 불가능, 비동기 워크플로 인프라 필수
