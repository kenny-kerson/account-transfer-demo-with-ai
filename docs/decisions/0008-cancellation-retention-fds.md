# ADR-0008: 취소·보관·FDS 정책 (Round 5 운영 부분)

- Status: Accepted
- Date: 2026-05-23
- Deciders: kenny.k
- Tags: policy, operations, retention, security

## Context

Round 5에서 결정된 운영·보안 정책 중 멱등성·동시성을 제외한 부분. (멱등성·동시성은 ADR-0010에 별도)

## Decision

### 취소 정책
- **완료 전 경과 상태(예: "이체중")에서도 사용자 취소 가능**
- 보상 트랜잭션·경합 처리 필요 (출금은 했지만 입금 전 취소 등)
- **백오피스는 완료 후 수동 조정** 가능 (TransferAdjustment 엔터티로 기록, ADR-0009 카테고리 E)

### 보관 기간 (국내 금융권 표준)
- **거래내역 10년** — 전자금융거래법
- **감사로그 5년** — 신용정보이용·보호법 등
- **운영로그 1년**
- 데모도 이 구조를 따르되 실제 저장은 축소된 테스트 데이터로 운용

### FDS (이상거래 탐지)
- **v1 제외**, 향후 구현 대상으로 식별
- 외부 FDS 시스템 위임 클라이언트 패턴이 적용될 영역 (ADR-0003 #6)
- v1에서는 차등 한도·인증수준으로 간접 통제만

## Rationale

- 취소: "이체중" 상태에서의 사용자 취소는 보상 트랜잭션·경합 시연 가치가 가장 큰 영역
- 보관 기간: 실제 보관 구조는 유지(파티션·아카이빙 정책 시연), 테스트 데이터만 축소
- FDS v1 제외: 외부 위임 클라이언트 패턴은 다른 시스템들로 충분히 시연됨

## Alternatives Considered

- **취소는 완료 후만, 진행 중 불가** — 거부됨. 보상 트랜잭션·경합 시연 가치 손실
- **사용자 취소 전면 금지, 백오피스만** — 거부됨. UX 부적합 + 시연 가치 약화
- **보관 기간 단일화 (전체 1년)** — 거부됨. 실제 법규 구조 학습 가치 ↑
- **FDS 자체 구현** — 거부됨. 도메인 깊이 부적합, v1 범위 초과

## Consequences

- (+) "이체중 취소" → 보상 트랜잭션·경합 처리·낙관락 시연 가치
- (+) 보관 기간 차등 → 데이터 라이프사이클 정책·아카이빙·파티션 정리 패턴 시연
- (+) FDS 외부 위임 식별 → 향후 확장 경로 명확
- (-) 취소 처리 = 상태머신 분기 ↑, TransferStateHistory 정확성 ↑
- (-) 다른 영역 정합성 검증 필요 (FDS 미적용 = 일부 시나리오 보호 약함, 데모임을 명시)
