# ADR-0006: 한도·수수료·가용시간 (Round 3)

- Status: Accepted
- Date: 2026-05-23
- Deciders: kenny.k
- Tags: policy

## Context

이체 정책의 기본 규칙: 누가 얼마까지·언제·수수료는 어떻게.

## Decision

### 이체 한도
- **이체 종류·인증수준별 차등 한도**
- 시스템 기본 한도 + 사용자 개인 한도 (시스템 한도 이하)
- 한도 정책·사용량은 별도 엔터티로 관리 (TransferLimitPolicy / UserTransferLimit / TransferLimitUsage — ADR-0009 이후 카테고리 B)

### 수수료
- 수수료 정책은 시스템에 포함 (관리 대상)
- **수수료 정책 평가·수수료 출금은 외부 시스템에 위임** (ADR-0003)

### 계정계 (원장 입출금)
- **외부 단위 시스템에 위임** — 이체 서비스는 출금/입금을 직접 처리하지 않고 계정계에 명령
- 본 시스템은 오케스트레이션 + 위임 클라이언트

### 가용 시간
- **24/7 실시간 (오픈뱅킹 모델)**
- 영업시간/대외계 시간대 구분 없음

## Rationale

- 차등 한도: 인증 수준이 약한 PIN-only 이체는 소액으로 제한, 추가 인증 시 한도 상향이 실무 표준 패턴
- 24/7 실시간: 시연 가치 ↑ (영업시간 분기 로직보다 외부 시스템 다운타임/지연 처리가 시니어 패턴에 더 적합)
- 수수료·계정계 외부 위임: 본 시스템 코어 도메인을 오케스트레이션에 집중

## Alternatives Considered

- **시스템 고정 한도** — 거부됨. 차등 한도가 인증 클라이언트와 자연스럽게 연결되어 시연 가치 ↑
- **수수료 정책까지 자체 구현** — 거부됨. 수수료 도메인 자체가 별도 단위 시스템 영역
- **은행 영업시간/대외계 시간대 구분** — 거부됨. 데모 단순화 + 24/7 실시간이 오픈뱅킹 표준
- **계정계 내장** — 거부됨. 원장은 별도 단위 시스템이 표준 (ADR-0003)

## Consequences

- (+) 차등 한도 → 정책 엔진 + 사용량 집계 시연 가치
- (+) 24/7 실시간 → 외부 시스템 비동기 응답·실패 처리 패턴이 항상 필요 (시연에 좋음)
- (-) 외부 위임 = 분산 트랜잭션 + 보상 처리 복잡도 (ADR-0010 멱등성·동시성과 연계)
- (-) 수수료 위임 호출 실패가 이체 흐름에 영향 (보상 시나리오 추가)
