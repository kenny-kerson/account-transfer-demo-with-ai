# ADR-0002: 프로젝트 성격 — 시니어 실무 수준 데모 + NFR 목표

- Status: Accepted
- Date: 2026-05-23
- Deciders: kenny.k
- Tags: architecture, nfr, scope

## Context

본 프로젝트는 "계좌이체 데모"라는 표현으로 시작되었지만, 사용자는 의도적으로 다음을 강조했다:

> "1번 서비스를 실제 프로덕션 환경에 구현하는 수준의 난이도를 가진 데모를 작성하는 것이 목적이야. 데모지만, 대용량 트래픽 / 동시성 / 트랜잭션 / 모니터링 등 시니어 엔지니어 수준의 실무작업이 반드시 포함되어야해"

따라서 단순 CRUD 학습 자료가 아니라 운영에서 발생할 실제 이슈(타임아웃·재시도·대사조정·멱등성·분산락 등)를 재현하는 학습·시연용이다.

## Decision

### 프로젝트 성격
- **은행/금융사 자체 인터넷·모바일 뱅킹 서비스**의 이체 도메인을 시뮬레이션
- **데모이지만 시니어 엔지니어 실무 수준 구현 필수**: 대용량 트래픽·동시성·트랜잭션·관측성을 모두 다룬다
- 모든 설계 결정은 "운영에서 발생할 실제 이슈 재현"을 우선 가치로 평가

### NFR 목표 (대형 은행 수준)
- **피크 처리량**: 5,000 TPS
- **응답시간**: P99 300ms
- **가용성**: 99.99% (월 다운타임 약 4.3분)

## Rationale

- "중형 뱅킹 수준(500 TPS / P99 1초 / 99.9%)"으로는 다층 락·샤딩·CDC·이벤트 기반 워크플로 등 시니어 패턴의 시연 가치가 약함
- 대형 은행 수준 NFR을 설정하면 자연스럽게 사용자별 샤딩·일자별 파티션·읽기 모델 분리·비동기 처리·서킷 브레이커·재시도 등 시니어 패턴이 정당화됨

## Alternatives Considered

- **중형 뱅킹 수준 NFR** — 거부됨. 시연 가치 부족
- **NFR 미정 (구현 단계에서 결정)** — 거부됨. 사용자가 명시적으로 추천 요청

## Consequences

- (+) 시니어 실무 패턴 시연 정당화
- (+) 데모지만 실제 트래픽 시뮬레이션·부하 테스트 의미 있음
- (-) 데모 복잡도 ↑ (단순 학습자에게는 진입장벽)
- (-) 구현 비용 ↑ (분산락·샤딩·관측성 인프라 모두 필요)
